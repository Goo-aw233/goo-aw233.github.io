获取微软商店应用的安装包

发表于 2022-07-13 更新于 2026-03-10 分类于 教程 ， Windows 阅读次数： 本文字数： 2k 阅读时长 ≈ 2 分钟
通过网站、Fiddler 工具或获取微软商店应用获取 Microsoft Store 应用的安装包下载链接，详细介绍操作步骤和注意事项。

store.rg-adguard

store.rg-adguard

  • 第一部分选择默认的 URL (link)
  • 第二部分输入 Microsoft Store 所对应的网页地址
  • 第三部分选择默认的 RP
  • 更新通道如下：Fast = 快速测试版 | Slow = 最快速测试版 | RP = 预发布版 | Retail = 正式版

获取商店链接

以微软电脑管家为例
从 Microsoft Store 点开应用详情页面，点击右方的“分享”，并在弹出的窗口中选择“复制链接”

完成后粘贴到网站的第二部分，然后点击 √ 即可查找适用的包
加载一会，你会看到各种各样的包

其中，

  • 后缀为 .BlockMap 的文件是文件属性，不需要下载
  • 应用程序本体一般是捆绑包（即 .AppXBundle 或 .MSIXBundle），内涵多个架构版本，必包含 x64，包含 ARM、ARM64 和 x86 其中之一，或部分，或全部）
  • 包名中带有 ARM、ARM64、x64 或 x86 字样的则对应的是你的系统架构，一般情况下选择 x64
  • 后缀为 .EAppX、.EAppXBundle、.EMSIX 和 .EMSIXBundle 的则是加密包，没有许可是无法直接安装的[1]
  • 应用运行需要运行库，所以你还需要下载与你系统架构对应的运行库，如果你不知道选哪个版本（不是架构）的运行库，则可以全部下载后逐个安装

    若你不知道你的 Windows 架构，请参阅：检查 CPU 与 Windows 的架构

获取应用程序包名

类似于 Microsoft Store，在商店内是搜索不到的

将应用从“开始”菜单拖出到桌面以创建快捷方式，右键它并选择“属性”，其中“目标类型：”或“目标(T)：”就是应用的包名

这时候回到网页，在第一部分选择 PackageFamilyName

第二部分输入包名（包名不包括后缀的 !App）

Fiddler

Fiddler v5.0.20202.18177 中文版 2020.10.29（已同步到资源列表）

首次使用 Fiddler

点击“取消”以解除 AppContainer 警告

在 Fidder 顶部工具栏点击“工具” > “选项…” > “HTTPS”，把“捕获 HTTPS 链接”和“解密 HTTPS 通信给勾选上”，这时候出现安装证书，点击“是”即可

在这俩弹窗点击 Yes 以继续安装 CA 证书

回到主程序，点击左上角的 WinConfig（若有警告请选择 No）
找到名为“Microsoft Store”的项，勾选它，并点击“保存更改”

获取下载链接

先在 Microsoft Store 下载应用（建议先退出无关应用避免有多余的链接产生）
回到主程序，当软件正在下载时列表会刷新
找到带有 tlu.dl.delivery.mp.m... 的链接并右键它，选择“复制” > “仅 URL”

粘贴到浏览器或其他下载器即可开始下载

获取微软商店应用

获取商店应用地址：GitHub 仓库 | Microsoft Store

关于获取商店链接和应用程序包名，请参阅：获取商店链接 | 获取应用程序包名

下载或获取下载链接

从左侧导航栏点击“微软商店”

  • 第一部分选择默认的“URL (链接)”
  • 第二部分选择默认的“正式版”
  • 第三部分输入 Microsoft Store 所对应的网页地址
    最后点击“查询链接”，即可开始搜索相关结果

其中，

  • 后缀为 .BlockMap 的文件是文件属性，不需要下载
  • 应用程序本体一般是捆绑包（即 .AppXBundle 或 .MSIXBundle），内涵多个架构版本，内含 x64，包含 ARM、ARM64 和 x86 之一或部分或全部）
  • 包名中带有 ARM、ARM64、x64 或 x86 字样的则对应的是你的系统架构，一般情况下选择 x64
  • 后缀为 .EAppX、.EAppXBundle、.EMSIX 和 .EMSIXBundle 的则是加密包，没有许可是无法直接安装的[1]
  • 应用运行需要运行库，所以你还需要下载与你系统架构对应的运行库，如果你不知道选哪个版本（不是架构）的运行库，则可以全部下载后逐个安装

    若你不知道你的 Windows 架构，请参阅：检查 CPU 与 Windows 的架构

点击包名即可使用内置的下载器开始下载包

推荐设置

  • 设置 > 商店与更新 > 查询链接方式
    • 使用“官方接口”，这会从 Microsoft Store 查询结果
    • 使用“第三方接口”，这会从 store.rg-adguard 查询结果
  • 设置 > 商店与更新 > 国家和地区
    关闭“使用系统国家或地区值”，并更改“商店区域”
    更改为合适的区域可以获取更多结果，例如针对不同地区推送不同的应用版本
  • 设置 > 下载管理 > 下载引擎方式
    如果你开启了传递优化，可以选择使用“传递优化”
    如果未开启传递优化，推荐使用 Aria2
    • 关于“传递优化”和“后台智能传输服务”，请参阅
      什么是传递优化？ | Microsoft Learn
      后台智能传输服务 - Win32 apps | Microsoft Learn
