VMware 虚拟机用 DMG 安装 macOS

发表于 2022-02-13 更新于 2026-07-08 分类于 教程 ， macOS ， 虚拟机 ， VMware ， Workstation 阅读次数： 本文字数： 3.3k 阅读时长 ≈ 3 分钟
本文详细介绍如何在 VMware Workstation Pro 中，利用 DMG 镜像文件、Etcher 工具和 VHD 虚拟硬盘，无需 U 盘即可引导并安装 macOS 虚拟机，涵盖从镜像下载、虚拟磁盘创建到写入和挂载的全过程。

前言

VMware 虚拟机安装 macOS

准备工具

Python
Etcher v1.4.9 （建议选带 Portable 的版本）
.DMG 镜像文件

使用 OpenCore 下载在线安装的 .DMG 镜像文件

macRecovery 是从 Apple 服务器下载镜像，下载速度因人而异

下载 OpenCore，选择带有 RELEASE.zip 标识的文件下载
将压缩包解压，打开到 \Utilities\macrecovery 文件夹，按住 SHIFT，并按下右键，选择“在此处打开 PowerShell 窗口”（Windows 11 或安装终端的 Windows 可以直接右键选择“在终端中打开”）
输入如下命令，以开始下载安装镜像。下载好的镜像存放在 macrecovery 或 com.apple.recovery.boot 文件夹下

如果是 macOS 或 Linux 则需要去掉 macrecovery.py 前的 py 并添加 ./
如 ./macrecovery.py -b xxx
更多 macOS 版本下载，请参阅：
https://github.com/acidanthera/OpenCorePkg/blob/master/Utilities/macrecovery/recovery_urls.txt
同目录下也有可用的 recovery_urls.txt

创建虚拟硬盘

右键“开始”菜单，选择“磁盘管理”（按下 WIN + R 以打开“运行”，输入 diskmgmt.msc 并回车）
在“磁盘管理”顶部菜单栏选择“操作” > “创建 VHD”

点击“浏览”，选择虚拟硬盘的存放位置，并起一个名字
虚拟硬盘格式可改为 VHDX，虚拟硬盘大小按需分配，可直接创建 16 GB 的大小一步到位，在线恢复镜像只需要 4 GB 即可

创建完成后，右键新建的虚拟磁盘，选择“初始化磁盘”

将磁盘格式初始化为 GPT 即可

保留“磁盘管理”窗口开启

写入虚拟硬盘

打开 Etcher，如果有更新弹窗选择 Skip，不要更新

点击 Select Image 并选择下载好的 .DMG 镜像
默认会选择虚拟磁盘，如果不是目标磁盘，请点击 Change 以选择正确的目标磁盘
完成后，点击 Flash! 即可开始

如果 Etcher 请求管理员权限，点击“是”即可
等待一下，写入过程还是很快的

如果遇到报错，则表示该镜像不可引导，需要更换一个

出现 Flash Complete! 即代表写入完成

回到磁盘管理
右键新建的虚拟磁盘，选择“分离 VHD”

在虚拟机中使用

在创建好的 macOS 虚拟机，点击“编辑虚拟机设置”
点击“添加”并选择“硬盘”，点击“下一步”

磁盘类型选择 SATA 即可，再选择“使用现有虚拟磁盘”

在浏览磁盘时，点击右下角的菜单栏选项，然后点击“所有文件 (.)”，并选择刚刚写入好的虚拟磁盘文件

选择“保持现有格式”即可，如果无法导入或无法开机，可选择备份文件后，选择“转换”后导入

完成后，启动虚拟机即可
